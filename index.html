Best Practice 1: クラウド資産管理システムの自社構築

PythonとAWS APIを活用し、グループ内のマルチアカウント環境におけるクラウドリソースとリスクの統合管理を実現しました。（N社）

課題・背景 グループ内の各種ゲームサービス等が個別のAWSアカウントで運用されており、セキュリティ部門からクラウドリソースの現状把握や脆弱性の統合管理が不可能な「ブラックボックス化」状態でした。
技術的詳細
  • 開発言語: Python
  • 連携技術: AWS API (EC2, ECS, RDS等のリソースAPI, AWS Inspector API)
  • 各AWSアカウントからの権限移譲を受け、全リソース情報を一元的に収集・パースするバッチ処理ロジックを実装。
主な取り組み
  • AWS Inspector APIと連動し、特定の「CVEコード」を入力するだけで、全アカウントにまたがる脆弱性の影響範囲を瞬時に検索できる機能を開発。
  • IAMアカウント管理、ネットワーク設定、バックアップ状態などのコンプライアンス要件を自動でチェックし、違反項目を抽出する自動点検プロセスを確立。
成果と意義 従来は管理の死角となっていたクラウド環境に対する完全な可視性(Visibility)を獲得しました。新たな脆弱性(0-day等)の発表時にも、迅速に影響度を把握し、即座に是正措置を講じることが可能な体制を構築しました。

Best Practice 1: 클라우드 자산 관리 시스템 자사 구축

Python 및 AWS API를 활용하여, 그룹 내 멀티 계정 환경의 클라우드 리소스 및 리스크를 통합 관리하는 시스템을 구축했습니다. (N사)

도입 배경 그룹 내 다수의 게임 등 서비스가 개별적인 AWS 계정으로 운영되고 있어, 보안 부서 차원에서 리소스 현황을 파악하거나 중앙 집중적인 리스크(취약점) 관리가 불가능한 블랙박스 상태였습니다.
기술적 상세
  • 개발 언어: Python
  • 연동 기술: AWS API (EC2, ECS, RDS 등), AWS Inspector API
  • 서비스별 AWS 계정의 접근 권한을 할당받아, API를 통해 전체 클라우드 리소스 데이터를 일괄 수집하고 파싱하는 로직을 직접 구현했습니다.
주요 수행 내역
  • AWS Inspector API를 응용하여, 특정 CVE 코드 입력 시 전체 계정을 대상으로 취약점 영향도를 즉각적으로 스캔하고 파악하는 기능을 개발했습니다.
  • IAM 계정 관리, 네트워크 구성, 백업 상태 등 필수 보안 컴플라이언스를 스크립트로 자동 체크하여 위반 사항을 점검하는 프로세스를 반영했습니다.
성과 및 의의 기존 클라우드 환경의 가장 큰 문제였던 가시성(Visibility) 부재를 완벽히 해결했습니다. 이를 통해 신규 취약점 발생 시 즉각적인 영향도 파악은 물론, 체계적이고 선제적인 클라우드 보안 리스크 관리가 가능해졌습니다.

Best Practice 2: オフィスセキュリティ点検システムの開発

物理セキュリティ（クリアデスク等）点検のプロセスをシステム化・自動化し、手作業によるレポート作成の手間を大幅に削減しました。（N社）

課題・背景 オフィスでのPCや機密文書の放置を点検する際、従来は全座席を撮影し、手動で全画像をダウンロードしてファイル名を座席番号に変更し、レポートを作成するという非効率的なプロセスが存在しました。
主な取り組み
  • 自社で運用中のシステムに「オフィスセキュリティ点検モジュール」を新規追加設計および開発。
  • 建物の各フロアの座席表をベースに、全座席をボタン型のUIでマッピング。
  • 対象座席のボタンをクリックし、「Pass / 違反」を選択。違反時は即座に証拠写真をアップロードする直感的なインターフェースを実装。
自動化の実現 点検完了後、違反履歴（アップロードされた写真と座席情報の紐付け）をシステム上で直ちに確認できるようにし、違反項目と該当写真のファイル名が自動的にマッピングされた「点検結果レポート」の自動抽出機能を実装しました。
成果と意義 従来は2フロアの点検に2時間要していましたが、本システムの導入により9フロアを3時間で点検可能になりました。これにより点検作業時間を約67%削減し、業務効率を300%向上させるという圧倒的な成果を達成しました。

Best Practice 2: 오피스 보안 점검 시스템 자체 개발

물리적 보안(클리어 데스크 등) 점검 프로세스를 전면 시스템화하여, 수작업에 의존하던 점검 및 레포트 작성 시간을 혁신적으로 단축했습니다. (N사)

도입 배경 오피스 내 노트북, 기밀 문서 등의 방치 여부를 점검할 때, 기존에는 각 자리를 사진으로 촬영하고 일일이 다운로드하여 정해진 규칙으로 파일명을 변경한 뒤 수동으로 레포트를 작성하는 비효율적인 절차를 거쳐야 했습니다.
주요 수행 내역
  • 자체 구축 시스템 내에 '오피스 보안 점검 모듈'을 신규 기획 및 기능 추가.
  • 건물의 각 층별 자리표(도면)를 기반으로 모든 좌석을 버튼 형태의 직관적인 UI로 구현.
  • 점검 대상 자리의 버튼을 눌러 점검 항목에 대한 '통과/위반'을 판정하고, 위반 시 현장에서 즉시 증적 사진을 업로드하도록 로직 설계.
자동화 구현 점검 완료 시 시스템 이력을 통해 위반 좌석과 사진을 즉시 확인할 수 있으며, 서버에 저장된 위반 사진의 파일명이 위반 항목과 자동으로 매핑되어 기입된 '점검 레포트 자동 추출 기능'을 완성했습니다.
성과 및 의의 기존 2개 층 점검에 2시간이 소요되던 업무를 9개 층 점검에 3시간으로 획기적으로 단축했습니다. 이는 소요 시간 67% 감소, 업무 효율성 300% (3배) 증가에 해당하는 엄청난 성과이며, 수작업 시 발생하던 파일명 오류 등 휴먼 에러를 원천 차단했습니다.

Best Practice 3: 統合セキュリティプラットフォーム(XDR)構築・高度化

SSL可視化ソリューションによる暗号化トラフィックの復号化と、各種セキュリティアプライアンスへのミラーリング構成により、全社的な監視体制を強化しました。

課題・背景 サイバー攻撃の高度化およびトラフィックの暗号化(HTTPS)が進む中、既存のファイアウォールやIPSだけでは脅威の可視化と包括的な分析が困難になっていました。
技術的詳細と
アーキテクチャ
  • インターネット接続トラフィックをSSL可視化機器(SWG+SSLVA)でPlain-textに復号化。
  • 復号化されたミラートラフィックをAPT NXやDLPに並行して転送する高度なネットワーク設計を導入。
主な取り組み
  • 老朽化したセキュリティ機器(FW, IPS)のシームレスなリプレイスを遂行。
  • ネットワーク機器、エンドポイントから発生するシステムログと検知イベントを統合収集し、XDRプラットフォームに連動させるパイプラインを構築。
成果と意義 サイロ化していた各種セキュリティログをXDR上で単一のEvent Streamとして相関分析することが可能となり、脅威の早期発見とリアルタイム警報システムの確立によるインシデント対応速度の飛躍的な向上を実現しました。

Best Practice 3: 통합보안플랫폼(XDR) 구축 및 보안 고도화

SSL 가시성 솔루션을 통한 암호화 트래픽 복호화 및 각종 보안 시스템으로의 미러링 구성을 통해 전사적 네트워크 모니터링을 강화했습니다.

도입 배경 사이버 위협이 고도화되고 웹 트래픽의 대다수가 암호화(HTTPS)됨에 따라, 노후화된 기존 방화벽과 IPS만으로는 내부로 유입되는 악성 행위에 대한 가시성 확보와 통합 분석이 불가능했습니다.
아키텍처 및
기술적 상세
  • 사용자의 인터넷 접속 트래픽을 SSL 가시성 솔루션(SWG+SSLVA)을 거쳐 Plain-text로 복호화하도록 네트워크 구성.
  • 복호화된 미러 트래픽을 APT NX와 DLP로 동시 전송하여 탐지율을 극대화하는 아키텍처 설계 및 구축.
주요 수행 내역
  • 노후화된 방화벽, IPS의 교체 및 정책 최적화 프로젝트 리딩.
  • 네트워크 시스템, DB, 단말에서 발생하는 방대한 이벤트 및 로그를 XDR 플랫폼으로 연동 및 파싱하는 데이터 파이프라인 구축.
성과 및 의의 파편화되어 있던 보안 이벤트를 XDR을 통해 통합 분석함으로써 실시간 경보 발령 및 선제적 침해 위협 대응이 가능해졌으며, 정보 보안 부서의 전반적인 업무 효율성과 가시성이 크게 향상되었습니다.

Best Practice 4: 網分離環境におけるコラボレーションツール接続統制

オープンソース(SQUID)を活用したプロキシを新規構築し、HTTPヘッダ識別を利用した高度なアクセス制御により網分離環境のセキュリティを強化しました。

課題・背景 網分離環境において特定グループウェアへのアクセスを許可する必要がありましたが、個人の私的アカウントでのログインを防ぎ、情報流出リスクを遮断する技術的制御が求められていました。
技術的詳細
  • OS/App: Redhat Enterprise 8.9 / Squid Cache 4.15
  • 制御ロジック: アクセス先URLと、企業ドメイン情報を含むHTTPヘッダ(AUTH等)の「AND条件」をProxyルールセットとして実装。
主な取り組み
  • 網分離区間にプロキシサーバーをゼロから単独で構築(寄与度100%)。
  • 既存ルールのマイグレーション、SSL証明書検証機能の適用、WebSocket通信のバイパス処理などを実施。
成果と意義 特定の企業アカウントのみログインを許可し、個人アカウントを完全に統制する環境を当該ソリューションメーカー初の事例として成功させました。これにより利便性を損なうことなく網分離環境からの情報流出の抜け穴を完全に塞ぎました。

Best Practice 4: 망분리 환경 업무협업도구 개인계정 접속 통제

오픈소스(SQUID) 프록시를 신규 구축하고, 기업 도메인 식별 헤더를 활용한 고도화된 접근 제어를 통해 망분리 환경의 보안을 강화했습니다.

도입 배경 인터넷이 차단된 망분리 환경에서 업무용 협업도구의 통신은 허용해야 했으나, 직원이 개인 계정으로 접속하여 회사 기밀 자료를 유출하는 행위를 차단할 기술적 통제 방안이 필수적이었습니다.
기술적 상세
  • 환경: Redhat Enterprise 8.9 / Squid Cache 4.15
  • 제어 로직: 목적지 URL과 기업 도메인을 식별할 수 있는 HTTP Header (AUTH 등)의 'AND 조건'을 프록시 룰셋으로 정교하게 구현.
주요 수행 내역
  • 망분리 구간 내 프록시 서버 단독 신규 구축 (프로젝트 기여도 100%).
  • SSL 인증서 검증(Trusted Root) 적용 및 WebSocket 기능 우회 방안 마련 및 테스트 수행.
성과 및 의의 오픈소스를 활용하여 개인 계정 로그인을 통제하는 환경을 제조사(솔루션 벤더) 최초 사례로 성공적으로 구축했습니다. 고도화된 기술적 통제를 통해 망분리 환경의 정보 유출 사고 위험을 근본적으로 제거했습니다.

Best Practice 5: Webシェル攻撃による侵害インシデント(IR)対応

脆弱性を突いたシステムへの侵入およびラテラルムーブメント(水平展開)攻撃を発見し、迅速な封じ込めから復旧・再発防止策までを主導しました。

インシデント概要
  • 発生時期: 2022年3月〜4月 (分析対象: サーバー10台, PC2台)
  • SmartEditor 2.1.2およびAtlassian Confluenceの脆弱性を悪用され、Webシェルおよび悪性ファイルがアップロードされた。
対応プロセス 1
(検知・初期対応)
  • 検知: Host IPSモニタリング中、内部IPセグメントに向けた不審なWebスキャン行為をいち早く検知。
  • 封じ込め: 被害の拡散を防ぐため、即座に感染が疑われるPCおよびサーバーのネットワークを隔離措置。
対応プロセス 2
(調査・復旧・強化)
  • 調査分析: 不審なプロセス調査とフォレンジック分析を実施し、C2サーバー(ロシア)との通信事実を特定。
  • 復旧・再発防止: 開発者のシステムアクセス統制ポリシーを見直し、全システムのパスワード強制変更および認証強化を実施。
成果と意義 巧妙なラテラルムーブメント攻撃を初期段階で遮断し、顧客データ等の致命的な情報流出を未然に防ぐことに成功しました。社内のインシデント対応マニュアルをアップデートし組織のレジリエンスを向上させました。

Best Practice 5: 웹쉘 공격 침해 사고 대응 (Incident Response)

알려진 취약점을 통한 시스템 침투 및 측면 이동(Lateral Movement) 공격을 조기에 발견하고, 즉각적인 격리부터 포렌식 분석, 보안 강화까지 전 과정을 수행했습니다.

인시던트 개요
  • 대응 기간: 2022년 3월 ~ 4월 (분석 대상: 서버 10대, 개인PC 2대)
  • SmartEditor 및 Atlassian Confluence 취약점을 악용하여 악성 웹쉘 4종 및 악성 파일 7종이 업로드됨.
대응 프로세스 1
(탐지 및 초기 대응)
  • 탐지: Host IPS 모니터링 중 내부 대역 IP로 향하는 웹 스캐닝 행위 등 비정상적인 특이 이벤트를 최초 발견.
  • 격리: 감염이 의심되는 PC 및 주요 서버의 네트워크를 즉각적으로 차단 및 격리 조치하여 피해 확산 차단.
대응 프로세스 2
(분석, 복구 및 강화)
  • 조사 분석: 포렌식 분석을 의뢰하여 C2 서버(러시아)와의 악성 통신 유발 시스템 식별.
  • 복구 및 강화: 내부 개발자의 시스템 접근 통제를 대폭 강화. 전체 시스템 패스워드 일괄 변경 및 보안 정책 재정립.
성과 및 의의 치명적일 수 있었던 내부망 침투 공격을 조기에 차단하여 실질적인 데이터 유출 피해를 완벽히 방어했습니다. 실전적인 침해 대응 역량을 증명하였으며, 사후 접근 체계 고도화를 이끌어냈습니다.
